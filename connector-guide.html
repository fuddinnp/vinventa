VMware ESXi

VMware ESXi is an enterprise-class, type-1 hypervisor for deploying and serving virtual computers. We use this connector to fetch devices from VMware ESXi.

Types of assets discovered from VMware ESXi:

The following types of assets are discovered by VMware ESXi:

Devices:

APIs:

vInventa uses the vCenter API to fetch information regarding virtual devices and ESX hosts.

Supported Versions:

Version Supported
VMWare Versions: 6.x, 7.x Yes

Note: While it may work with other versions, this connector has been tested with the versions marked as supported. In case you have versions which are not listed above or are not functioning as expected, please contact support@vinventa.com for immediate assistance.

VMware Connector Deployment Pre-requisites

The following fields are required for connector deployment:

I. ESXi Host Name

II. ESXi Username & Password

If you do not have this information available then go to the VMware ESXi Configuration section on page 8.

VMware ESXi Connector Settings in vInventa

From the main menu, go to the Connectors page and select the connector: VMware ESXi.

Click VMware ESXi Tile

Click Add Connection

Instance Profile Tab

The Instance profile tab is used to add in the information required to connect to VMware ESXi.

Field Default Description
Instance Name* None Uniquely identifies a connector instance for any particular connector (allowed characters 1-30)
Description None Add a description of the connector instance here (allowed characters 300 max)
Connector Label None Additional label/tag for a connector instance (allowed characters 1-30)
User Name* None Create a unique name for identification of a user (Please refer to VMware details)
Password* None Use alphanumeric and special characters to create a strong password (Please refer to VMware details)
Hostname* None Assign a label to a connected host device (Please refer to VMware details)

* Mandatory fields must be configured

Connector Settings Tab

This tab is used to configure each connectors optional configuration settings such as schedules, delete option and more.

Field Default Description
Use system default True By default, system will use global discovery schedule. The settings can be changed under Configuration> Discoveries
Use this schedule    
Day for scheduling discovery cycle False User can specify the day for scheduling a discovery cycle
Time for the selected day 12:00 PM User can specify the time for selected day
Number of hours between each discovery cycle None User can specify the number of hours between each discovery cycle
Time for initial discovery cycle None User can specify time for initial discovery cycle

Delete Assets:

This setting is used to define the conditions to delete discovered assets

Field Default Description
Use system default True By default, system will use global discovery Retention Policy. The settings can be changed under Configuration> Retention
Use Custom    
Delete Assets not discovered in the last False Specify the number of day(s), week(s), and month(s). Assets not discovered from this particular connector instance will be deleted if not discovered in the specified number of day(s), month(s), or week(s)

Discovery Settings Tab

This tab is used to define discovery conditions such as, assets and attributes, to be discovered by the connector

Settings Default Description
Powered on machines False If checked, only machines which are turned on will be fetched

VMware ESXi Configuration

Creating a User in vCenter

1. Log in to the vCenter as an administrator

2. Click Menu > Administration

3. Navigate to the menu on left

4. Click Users and Groups.

  • Under the Users tab, from the Domain field, select the domain
  • Click Add User

5. Fill in the relevant details.

6. From the left menu, click Global Permissions and add a 'Read-only' permission to the created user.

7. Select Propagate to children

8. Click OK.
